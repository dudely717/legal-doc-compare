# 综合总结输出结构模板

严格按以下顺序输出，不得调整章节顺序。

文件标题：**[项目名称-交易/融资轮次名称]总结**（作为 .docx 文件标题，不编号）

```
1. **交易方案。**
   > 本次[X]轮融资锁定[投前/投后]估值[X]元（前轮投后为[X]元，本次投前[高于/等于/低于]前轮投后），本次投资人合计投资[X]元，对应取得[X]%投后股权：
   > - [投资方A]：投资[X]元，取得投后[X]%股权
   > - [投资方B]：投资[X]元，取得投后[X]%股权
   > 我方（[客户名称]）股比将自前轮投后[X]%下降为本轮投后[X]%。
   > ***[客户股比变化确认及估值比较结论（溢价/平价/折价）]{.underline}***

2. **[核心条款变化1]。**
   > （变化内容摘要）
   > ***[决策问询]{.underline}***

3. **[核心条款变化2]。**
   > （逐项说明）
   > ***[逐项决策问询]{.underline}***
   > 若存在更多核心条款变化，重复本节直至覆盖全部

4. **[其他变化，如"董事会/观察员/核心员工"]。**
   > ***[如无实质影响，用"理解对[客户方]无影响，还请知悉"收尾]{.underline}***

5. **其他。**
   > 以下内容与前轮一致，无实质修订，还请知悉：
   > - [XX权利]与前轮一致
   > - [YY权利]与前轮一致
   > 性质相近的权利可合并为一行（如"优先购买权、共同出售权机制与前轮一致"）

6. **待跟进事项。**
   > ① [事项描述及建议]
   > ② [事项描述及建议]
   > ***[跟进承诺]{.underline}***
```

**注意事项**：
- **第6节"其他"**：仅列无实质修订的内容；性质相近或属于同一机制的项目合并为一行，不逐条重复；不需要决策问询
- **第7节"待跟进事项"**：仅列需要进一步沟通确认、补充材料或客户决策的事项，与"其他"严格分开
- 若有多轮投资人对比，用"前轮：……本轮：……"格式并列呈现
- 所有 `***[...]{.underline}***` 标记的内容，必须放在相应条款说明之后，紧接决策问询
- 输出时调用系统工具将 Markdown 转换为 .docx 文件并提供下载
